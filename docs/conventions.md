# Conventions

`AssistOps Free`는 포트폴리오 프로젝트이므로 구현 범위, 기술 선택, 문서 표현을 명확하게 유지합니다.

## 언어 규칙

- README와 docs 문서는 한국어로 작성합니다.
- 코드, 파일명, 폴더명, 변수명, 함수명, 클래스명은 영어를 사용합니다.
- API endpoint는 영어를 사용합니다.
- 설정 키와 환경 변수 이름은 영어를 사용합니다.
- Git commit message는 영어로 작성합니다.

## 문서 작성 규칙

- 아직 구현되지 않은 기능은 `예정`, `목표`, `향후 구현`으로 표시합니다.
- README와 docs에서 구현 완료 여부를 과장하지 않습니다.
- 목표 아키텍처와 현재 구현 상태를 분리해 작성합니다.
- 포트폴리오 설명에 필요한 의사결정 이유를 간결하게 남깁니다.

## 기술 선택 원칙

- 유료 SaaS/API에 필수적으로 의존하지 않습니다.
- 유료 AI API를 핵심 기능의 필수 구성으로 두지 않습니다.
- 로컬 실행 가능성을 우선합니다.
- 오픈소스 또는 무료로 재현 가능한 도구를 우선 검토합니다.
- 관리형 클라우드 서비스가 필요해지는 경우에도 로컬 대체 경로를 먼저 정의합니다.

## 코드 작성 원칙

- 기존 프로젝트 구조와 프레임워크 기본 설정을 존중합니다.
- 불필요한 추상화나 과한 초기 설정을 피합니다.
- 기능이 추가될 때 테스트와 문서도 함께 보강합니다.
- API, UI, 인프라 변경은 현재 구현 상태와 향후 목표를 구분해 기록합니다.
